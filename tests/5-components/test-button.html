This page verifies the functionality of the .c-button component and its modifiers defined in the 5-components Sass layer.

Development note: the helper styles on this page exist only to make button boundaries, spacing, full-width behavior, and wrapping behavior easier to inspect.

Base button

Expected behavior: The base button should render as an inline-flex control with centered content, visible padding, rounded corners, no text underline, and stable alignment whether it is rendered as a link or a button element.

Base button as link

Filled action variants

Expected behavior: The primary, secondary, and section CTA filled variants should share the same base filled visual treatment. On hover and focus-visible, each should switch to the hover token defined by the active theme.

Section CTA variant

Expected behavior: The section CTA modifier should expand to the full inline width of its parent container. This test simulates a section action area below a heading or below a content block. The button should fill the available width rather than shrink to its label width.

Icon composition

Expected behavior: Buttons using .c-button--with-icon should align the icon circle and label on one line with consistent spacing. The icon circle should remain circular and should not stretch. The secondary icon-circle hover behavior should also be visible on hover and focus-visible.

Disabled state

Expected behavior: Disabled buttons should appear visually muted and should not behave like interactive controls. This verifies both the native :disabled state and the [aria-disabled="true"] pattern.

Small-screen behavior

Expected behavior: On very narrow screens, normal buttons may be forced by the helper wrapper to fill the available width, while the section CTA should remain intentionally full-width by its own modifier. Once the viewport reaches the component’s min-width breakpoint, standard buttons should prefer content width more explicitly, but the section CTA should continue to fill its parent container.

White-space stress test

Expected behavior: Because the button uses white-space: nowrap, a very long label should stay on one line. This test is useful for spotting overflow risk and deciding whether your real content stays within sensible button lengths.
