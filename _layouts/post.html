--- layout: default --- {% comment %} Post Layout Purpose: Render long-form article pages used for projects, essays, research notes, and documentation posts. Structure: - Post page container - Optional cover image (same readable width as header/content) - Article wrapper - Post header (same readable width as cover/content) - Optional tag list - Post title - Optional subtitle - Header row (metadata; share is the no-TOC mobile fallback) - Mobile/tablet TOC toolbar and panel - Post body content - 3-column content layout (desktop) - Left column (desktop sticky share rail) - Main column (prose content) - Right column (desktop Table of Contents) - Single-column layout (mobile/tablet) - Prose content - Desktop TOC hidden - Post navigation (full-width) - Related posts section (full-width) - Back-to-top control Layout: - Cover image, header, and prose share the same readable width - Content area uses a symmetric 3-column system: | left column (share rail) | prose | right column (TOC) | - Prose width is constrained via o-prose - Left and right columns remain equal to preserve visual centering Table of Contents: - Shared list generation is delegated to toc.html - Mobile/tablet wrapper is delegated to post-toc-mobile.html - Desktop wrapper is delegated to post-toc-desktop.html Architecture: - Layout layer of the site - Uses ITCSS + BEM naming - Content layout handled by c-post-layout - Reusable UI fragments are delegated to includes - Tag links use the shared c-tag component Accessibility: - Uses semantic article and header elements - TOC wrappers use nav landmarks with labels - Mobile TOC toggle uses aria-expanded and aria-controls - Side layout columns hold the share rail and TOC; neither is aria-hidden Data dependencies: - page.title - page.subtitle - page.image - page.image_alt - page.image_width - page.image_height - page.image_srcset - page.image_sizes - page.image_position - page.tags - page.share - page.toc - content {% endcomment %}
