# Typography

## 1. The Core Terminology

* **Typeface vs. Font:** A *typeface* is the design (e.g., Roboto), while a *font* is the specific file or style (e.g., Roboto Bold 16px). Think of a typeface as a song and a font as the MP3 file.
* **Serif:** Small strokes at the ends of letters (e.g., Times New Roman). Feels traditional and formal.
* **Sans-Serif:** No extra strokes (e.g., Arial). Feels modern, clean, and friendly.
* **Monospace:** Every character is the same width. Essential for code to ensure alignment.

## 2. Choosing and Pairing Fonts

* **Personality Matters:** Choose fonts that match your brand. Serifs lean toward "expensive" or "serious," while Sans-Serifs lean toward "tech" or "approachable."
* **Check Weights:** For complex UIs, choose a typeface with many weights (Light, Regular, Medium, Bold, Black) to create a clear hierarchy.
* **True Styles:** Avoid "faux" bold/italics generated by browsers. Ensure your font files include dedicated `italic` and `bold` variants for better legibility.
* **Pairing Tools:** Use [FontPair](https://fontpair.co/) or [Typewolf](https://www.typewolf.com/) to find combinations that work.

## 3. Sizing and Units

* **Base Units:** Use `px` as a foundation for your type system, but use relative units like `em` or `rem` for the actual implementation to ensure scalability.
* **Relative Scaling:** `1.5em` is 1.5x the parent font size. This keeps your layout proportional if the base size changes.
* **Hierarchy:** Use distinct size jumps between levels (e.g., H1, H2, Body) so the user can instantly distinguish headers from content.

## 4. Spacing for Readability (The "Golden Rules")

* **Line Height (Leading):** Essential for vertical legibility. A good default for body text is `1.5`. Use lower values for headings.
* **Line Length (Measure):** The ideal line length is **60–75 characters**.
* *Dev Tip:* Use the `ch` unit in CSS: `max-width: 65ch;`


* **Letter Spacing (Tracking):**
* **Large Text:** Slightly decrease spacing (e.g., `-0.02em`) to make headings look tighter.
* **Small Text:** Slightly increase spacing to prevent letters from "clumping."


* **Kerning:** Enable built-in font kerning with `font-kerning: normal;`.

## 5. Implementation & Optimization

* **Font Loading:** Use hosted services like **Google Fonts** (fast/free) or **Adobe Fonts** (high quality) via a simple `<link>` tag.
* **Legibility Check:** If your text is difficult to read at 14px or in a long paragraph, the font is too "decorative." Stick to simple faces for body copy.
* **Font Smoothing:** On MacOS/Safari, use `-webkit-font-smoothing: antialiased;` for a cleaner look, but use it sparingly as it can reduce contrast on small text.

## Summary Checklist for Developers:

1. [ ] Is the body font easy to read at small sizes?
2. [ ] Does the line length stay under 75 characters?
3. [ ] Is the `line-height` at least 1.4 or 1.5?
4. [ ] Did you include the true bold and italic font weights?
5. [ ] Is there a clear visual difference between your H1, H2, and Body text?
