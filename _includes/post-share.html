{% comment %} Post Share Component Purpose: Provide configurable sharing actions for blog posts. Features: - Share actions are driven by _data/share.yml - Supports both links and buttons - Social share popups for supported platforms - Email sharing - Print to PDF (browser print) - Native share API with clipboard fallback - Optional overflow disclosure for compact contexts Architecture: - ITCSS + BEM naming - Component prefix: c-post-share - Shared primitive: c-icon-link - Utility: u-list-reset - JS hooks: js-*, data-overflow-* - Content/configuration comes from site.data.share - Per-item rendering is delegated to post-share-item.html so inline and overflow actions render identically - Filtering and ordering logic remains in this include Accessibility: - Per-item accessibility is delegated to post-share-item.html - The overflow toggle uses aria-expanded and aria-controls - The overflow panel is hidden via the `hidden` attribute when collapsed - Clipboard fallback feedback is exposed through a polite live region Params: - collapse (optional, boolean): when true, only the first available "platform" action (x, linkedin, pinterest, or email - whichever comes first and is available) plus the pinned Print and Native actions render inline; any remaining platform actions render inside a hidden overflow panel revealed by an "ellipsis-horizontal-circle" toggle. Used by the mobile/tablet TOC toolbar to keep it compact. Omit (or false) to render every available action inline (used by the desktop share rail and the no-TOC header fallback). Dependencies: - site.data.share - page.title - page.url - page.image (optional; required only for image-based share actions) - _includes/post-share-item.html - _includes/icons/ellipsis-horizontal-circle.svg - assets/js/post-share.js - 5-components/_icon-link.scss - 5-components/_post-share.scss Usage: {% raw %}{% include post-share.html %}{% endraw %} {% raw %}{% include post-share.html collapse=true %}{% endraw %} Notes: - Reorder, remove, or add actions in _data/share.yml - The pinned actions are identified by id ('print', 'native'); all other ids are treated as "platform" actions for the collapse split - If an action requires page.image and the page does not provide one, that action is skipped automatically {% endcomment %} {% assign share_items = site.data.share %} {% assign available_items = share_items | where_exp: "item", "item.requires_image != true or page.image" %} {% assign collapse = include.collapse | default: false %} {% if collapse %} {% assign pinned_items = available_items | where_exp: "item", "item.id == 'print' or item.id == 'native'" %} {% assign regular_items = available_items | where_exp: "item", "item.id != 'print' and item.id != 'native'" %} {% assign visible_regular = regular_items | slice: 0, 1 %} {% assign overflow_items = regular_items | slice: 1, regular_items.size %} {% assign visible_items = visible_regular | concat: pinned_items %} {% else %} {% assign visible_items = available_items %} {% assign overflow_items = '' | split: ',' %} {% endif %}
