{% comment %} Social Links Component Purpose: Render reusable social platform links plus the CV action, in a consistent priority order, with an optional overflow disclosure. Structure: - Ordered list of available items (CV + social platforms) - Optional overflow disclosure: a toggle button plus a hidden panel holding any items beyond `collapse_after` Architecture: - Reusable leaf include used by header, footer, and mobile menu - Output is markup only; layout belongs to the parent container - Per-item markup is delegated to social-link-item.html so visible and overflow items render identically - Uses c-icon-link and c-social-links element classes - JS hooks: js-* / data-* (overflow disclosure only) Accessibility: - Per-item accessibility is delegated to social-link-item.html - The overflow toggle uses aria-expanded and aria-controls - The overflow panel is hidden via the `hidden` attribute when collapsed Params: - collapse_after (optional, integer): when set and the number of available items exceeds this value, only the first `collapse_after` items render inline; the rest render inside a hidden overflow panel revealed by an "ellipsis-horizontal-circle" toggle button. Omit (or 0) to always render every available item inline (used by the footer and mobile menu, which wrap and center instead of collapsing).
