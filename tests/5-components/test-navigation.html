This page verifies the functionality of the navigation classes defined in the 5-components Sass layer.

Base navigation structure

Expected behavior: The navigation list below should render as a horizontal flex row with aligned items and no default list bullets because .c-nav__list works together with .u-list-reset.

  • Home
  • Projects
  • About
  • Contact

Desktop navigation variant

Expected behavior: The desktop variant uses .c-nav--desktop. Below the layout breakpoint it should be hidden. At and above the layout breakpoint it should become visible.

Desktop nav test:

  • Home
  • Projects
  • Writing
  • Contact

Resize the browser width to confirm the desktop navigation visibility rule.

Active and current-page states

Expected behavior: Links with .is-active or aria-current="page" should use the active color. In the non-mobile navigation style, they should also show a small circular marker near the bottom.

  • Overview
  • Projects
  • Research
  • Contact

Mobile navigation variant

Expected behavior: The mobile variant should stack links vertically, stretch items to the available width, align link text to the left, and show separators between links. Its active link should not render the circular marker used by the desktop/default style.

  • Home
  • Projects
  • Publications
  • Contact

Long-label wrapping behavior

Expected behavior: In the default or desktop-style navigation, links should stay on a single line because .c-nav__link uses white-space: nowrap. In the mobile variant, long labels should be able to wrap because the mobile modifier changes that to white-space: normal.

External-link behavior

Expected behavior: The external link below should look visually like the other navigation links. In your real include, external links also receive target="_blank" and rel="noopener noreferrer".

  • Projects
  • External CV

Header mobile toggle

Expected behavior: The toggle button should render as a circular icon button. On hover it should change color/background. The version with aria-expanded="true" should show the expanded state styling. At and above the layout breakpoint, the toggle should become hidden.

Collapsed state: aria-expanded="false"

Expanded state: aria-expanded="true"

Tip: Resize the browser window to confirm that .c-nav--desktop and .c-nav-toggle switch visibility at the layout breakpoint.
